# Start Here

Welcome. The goal of this workshop is to leave with a working, reviewable, and
replication-oriented research repository, not just a folder of notes.

## Beginning of Day 1

1. Fork this repository on GitHub.
2. Clone your fork locally:

   ```bash
   git clone https://github.com/YOUR_USERNAME/AC4E_Pavia.git
   cd AC4E_Pavia
   ```

3. Follow [`SETUP.md`](SETUP.md).
4. Read or bookmark the full [`GUIDE.md`](GUIDE.md). Use the split chapters in
   [`guide/`](guide/) when you want a topic reference.
5. Open this folder in your chosen agent:
   Codex app, Claude Code app, Cursor, a CLI coding agent, or another compatible tool.
6. Ask the agent a read-only question:

   ```text
   Read START_HERE.md, GUIDE.md, AGENTS.md, and days/day1.md. Do not edit files.
   Tell me what I should do first, which files I should avoid, and which command
   verifies the setup.
   ```

## What You Will Produce

- `docs/project_brief.md`
- `docs/research_design_memo.md`
- `docs/data_source_map.md`
- `notes/context_patterns.md`
- `notes/orchestration_log.md`
- one reusable skill or logged skill use
- one subagent/custom-agent artifact or logged delegated run
- `agent-harness/autonomous_agent_risk_card.md`
- `replication/README.md`
- a short presentation outline
- a 30-day adoption plan

## Working Principle

The agent can help you move quickly, but you own the research claim. Human review,
tests, clean-run instructions, and replication evidence are not optional extras.
